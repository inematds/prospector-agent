Uma agência de sites de uma pessoa só, no automático

Descobre negócios bem avaliados com site ruim, redesenha a página, publica e manda a proposta — orquestrador determinístico + IA só onde precisa, operado pelo Telegram.

Começar o guia Ver no GitHub 
# a esteira, do começo ao fim
descoberto
  → qualificado     # julga "site ruim?" (IA)
  → redesenhado     # página premium (IA)
  → publicado       # git → GitHub Pages
  → [ PORTÃO ]      # aprova no Telegram (ou --auto)
  → enviado         # proposta via Resend
  → respondido → fechado
O que é

Uma esteira que roda achou → refez → publicou → ofertou

Semi-autônoma: as fases pesadas rodam sozinhas, e você mantém o controle do envio. Baseada no plugin prospector-de-sites (usado como referência), reconstruída como serviço operável.

⛏️ Esteira completa

Descobre, qualifica, redesenha, publica e envia — cada fase é um estágio de uma máquina de estados que sobrevive a reinício e é idempotente por lead.

🧠 IA só onde precisa

Fila, portão e estados são Python puro e testável. O modelo (claude -p) entra só pra julgar o site e redesenhar a página — o resto é código, sem gastar token.

🔌 Plugável e portável

A descoberta troca de navegador para API por config. A mesma base roda local, híbrido em VPS ou tudo em VPS — sem reescrever nada.

Como funciona

O ciclo de vida de um lead

Cada seta é uma transição idempotente: se já foi feita, pula. O único ponto de humano é o portão antes do envio — e ele some quando o pedido vem com --auto.

descoberto→ qualificado→ redesenhado→ publicado→ [ portão ]→ enviado→ respondido→ fechado

Ramos: sem site / site bom / sem e-mail → descartado. Falha de rede/API → erro (alerta no Telegram, com /retry).

M1 · Fundação — o núcleo e o dry-run

Config, Store SQLite, máquina de estados e a CLI que avança um lead pela esteira sem publicar nem enviar.

python -m pytest -v                 # suíte do núcleo
python -m prospector demo --dry-run --auto
# → descoberto → qualificado → redesenhado → publicado → enviado
2

M2 · Descoberta + Qualificação

Acha candidatos no Google Maps (nota ≥ 4.7, com site) e julga quais têm site ruim, extraindo e-mail e WhatsApp.

prospector prospectar nutricionistas Bauru
# → leads qualificados salvos no prospector.db
3

M3 · Criação + Publicação

Redesenha a página premium (mantendo logo, cores e conteúdo reais) e publica no GitHub Pages com HTTPS.

prospector publicar <slug>
# → https://usuario.github.io/prospector-sites/<slug>/
4

M4 · Envio + Telegram (esteira completa)

Manda a proposta via Resend, com o portão de aprovação no Telegram. O bot opera tudo; o --auto pula o portão.

# pelo bot do Telegram:
/prospectar nutricionistas Bauru --auto
# fases correm sozinhas; o bot notifica cada envio
O núcleo

Controle determinístico, em código

O coração do sistema não é o agente — é a máquina de estados e o portão. Ficam em código simples e testável; a IA é chamada só dentro de dois estágios.

# prospector/maquina_estados.py — transições válidas
TRANSICOES = {
  DESCOBERTO: {QUALIFICADO, DESCARTADO},
  QUALIFICADO: {REDESENHADO, DESCARTADO},
  REDESENHADO: {PUBLICADO, ERRO},
  PUBLICADO: {AGUARDANDO_APROVACAO, ENVIADO, ERRO},
  AGUARDANDO_APROVACAO: {ENVIADO, DESCARTADO},
  ENVIADO: {RESPONDIDO, ERRO},
  RESPONDIDO: {FECHADO, DESCARTADO},
}
# o portão A+B, em uma linha
if lead.sem_portao or aprovado_no_telegram(lead):
    enviar(lead)          # Resend
else:
    aguardar_aprovacao(lead)   # pede OK no Telegram

# dry-run: roda a esteira sem publicar nem enviar
$ python -m prospector demo --dry-run --auto

           → publicado → enviado
